Lessons & process notes

What worked

  • Sequential per item. One codex at a time avoids the silent-fail mode where concurrent codex dispatches drop stdout.
  • Cross-family review caught a real bug. The bitnami URL would have shipped runnable-broken code to learners. Gemini caught it on the diff. Without the cross-family gate, this lands.
  • APPROVE_WITH_NITS held, not auto-merged. The recent fix (post day3-388 ab-discuss) preserved the distinction from APPROVE. The 1.4 case correctly stopped at the orchestrator for triage.
  • Round-2 prompt explicitly forced diff-reading. Per feedback_gemini_hallucinates_round2_approvals.md — gemini round-2 was told to curl-verify, not approve from description. That's the only reason this round-2 is trustworthy.

What to carry forward

  • Bitnami URL hazard. Any K8s module with helm repo add bitnami needs the chart-index URL https://charts.bitnami.com/bitnami (or OCI form), not github.com/bitnami/charts. Worth a one-line verifier check.
  • Fix-up dispatch pattern. Reuse the existing worktree, focused prompt with quoted reviewer feedback, force-fix only the flagged delta. Took ~3 min vs. a full re-rewrite.
  • Codex push verification. Verified git ls-remote origin <branch> after fix-up per feedback_codex_push_silent_failure.md. The push reached origin; this should be table stakes for any codex dispatch that pushes.
  • HTML-first artifacts. This report itself is the first artifact under the new HTML-default policy (per Thariq's article 2026-05-08). Replaces what would otherwise have been an unread session-state markdown table.
