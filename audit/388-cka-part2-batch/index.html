Lessons & process notes

What worked

  • Sequential pipeline held the line. One codex dispatch at a time avoided the silent-fail concurrency mode (concurrent dispatches drop stdout). 5h45m for 29 modules at ~11.9 min/module is a new throughput record for the project.
  • Claude review fallback saved the run. Gemini rate-limited at module 8/29 and never recovered. The review_fallback_to_claude path activated for all 22 remaining modules. Without it, the pipeline would have halted at 2.8 and required manual orchestration. Cross-family rule was still satisfied (codex writes, claude reviews).
  • Auto-merger v3 eliminated the NITS backlog. 21 PRs received APPROVE WITH NITS and were automatically merged by the script. What previously required an orchestrator pause and manual decision now runs unattended. The three-version iteration (v1→v2→v3) completed within 3 modules.
  • Contract + verifier alignment from PR #1025 prevented the duplicate-title bug from earlier batches. The T0 gate was hit by every single module with no verifier failures during the entire marathon.
  • Overnight autonomous mode worked end-to-end. No human check-ins were needed between launch and the final commit. The pipeline's milestone commits (4/29, 8/29, 15/29, 22/29) provided durable state for any interruption recovery.

What to fix

  • Gemini rate-limit handling needs backoff, not immediate fallback. All 22 fallbacks were caused by a single rate-limit storm with no retry delay. A 60–120s exponential backoff before triggering review_fallback_to_claude would recover gemini reviews in most cases and preserve the true cross-family guarantee.
  • Codex meta-diction leaks in reader-facing prose. Claude reviewers flagged "protected diagnostic asset", "original lesson", "preserves the original preparation-failure map" in 5.2 and others. These pipeline-internal terms ship to learners. The writer prompt must explicitly ban contract diction in output prose (see feedback_codex_prose_meta_diction_leak.md).
  • Source annotation regressions in 5+ modules. Reviewers consistently flagged that inline source annotations (— Backs the Kubernetes network model…) were stripped to bare hyperlinks. The writer prompt should explicitly preserve annotation format for sources that carry them in the original.
  • Claude review hard_timeout=300s too tight for large modules. 5.7 Logging & Monitoring (5,085 words) exceeded the 300s hard timeout. The review agent needs at least 450–600s for modules at the upper end of the word-count range, or the review task should be chunked.
  • Auto-merger should log its merges to the JSONL. The pipeline log has no auto_merged events — you can only infer merges from the absence of a merged event after merge_held_nits. Adding a {"event": "auto_merged", "pr": N} entry would make future audit reports more precise.
