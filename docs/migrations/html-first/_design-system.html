§3 Render-And-Review Contract

Every committed HTML artifact MUST be renderable by lightpanda:

lightpanda fetch --dump markdown http://127.0.0.1:8989/<path>

Agents reviewing HTML PRs use this command to read the artifact semantically. Lightpanda requires http://, so serve the repo root first, for example .venv/bin/python -m http.server 8989 --bind 127.0.0.1 when the venv is available.

If lightpanda's markdown rendering of your HTML drops critical information, restructure the HTML — don't add a sidecar.
