The bug that almost shipped (gemini-pro caught it)

Because GrokAdapter.build_invocation lets caller overrides win over mode defaults, read-only bridge sessions would have received write tools + auto-execution. Round-1 NEEDS CHANGES. Fix 298713b7 branches on mode.

The hermes bug we discovered

hermes -z - (stdin marker) is interpreted as "no prompt, project-introspection mode" and ignores piped content. Empirical proof:

echo "Reply OK" | hermes -z - -m grok-4   →  "Workspace: primary on... | pipelines v2: 567 modules"
hermes -z "Reply OK" -m grok-4              →  "Reply OK" ✓

Fix: pass prompt via argv positional, not stdin. ARG_MAX ~1 MB on macOS, fits review prompts (~10 KB) comfortably. Applied consistently in both the adapter and scripts/dispatch.py:dispatch_grok().
