Contract changes

Cold-start sequence (CLAUDE.md)

Numbered 6-step curl sequence: /api/briefing/session?compact=1 → git status --short → ls docs/decisions/pending/ → /api/session/current → optional /api/activity?limit=30 → fallback chain. On API-down, local steps 2-3 still run (they don't depend on the API), then fall back to STATUS.md → handoff → MEMORY.md → CLAUDE.md.

New endpoint contract: /api/orient

{
  "version": 1,
  "primary": { "action", "headline", "module_key", "endpoint", "reason", "phase" },
  "alternatives": [ ...up to 3 of the same shape... ],
  "blockers": [...],
  "alerts": [...],
  "workspace_dirty": bool,
  "freshness_state": "fresh|stale|refreshing",
  "generated_at": float
}

Pure projection over get_or_build_session_briefing snapshot. No new compute. 5s cache TTL. Returns primary.action == "idle" when action_rows is empty.

_check_url in verify_module.py

HEAD-fallback tuple is now (403, 404, 405, 501) (was (403, 405, 501)). GET-retry has its own try/except: on raise after a bot-block-pattern HEAD, falls through to lightpanda. HEAD itself raising still short-circuits to "fetch_failed" (original network-failure semantics preserved). Lightpanda fallback degrades gracefully if binary missing.
