agy (Antigravity CLI) facts established

Install + binary surface
  • Install: curl -fsSL https://antigravity.google/cli/install.sh | bash → ~/.local/bin/agy (version 1.0.0)
  • Binary name: agy (not antigravity). PATH is auto-updated in shell rc files by the installer.
  • Headless mode: agy -p "PROMPT" — positional argument only, stdin (echo "X" | agy -p) does NOT work.
  • Auth: stored somewhere we haven't found (no ~/.config/agy, no ~/.agy, no Keychain entry for "agy" or "antigravity"). Probably server-side via Google account OAuth.
  • State files: only ~/.cache/antigravity/staging (from install). No conversation/session files on disk.

Fixed in two layers: (a) dispatch_smart.py forces mode="danger" for --agent agy (matches the existing codex force-danger pattern at line 347), (b) the adapter unconditionally appends the flag regardless of mode (belt-and-suspenders for direct runner.invoke callers).
  • N1 — stderr_excerpt polluted on happy path: codex was always populating it with a "agy active model" informational note, which broke the documented contract that stderr_excerpt is None when no diagnostic output. Fix: removed the informational note + the unused _MODEL_HINT_ENV constant.
  • N2 — missing test for returncode=0 + stdout="": added test_parse_response_empty_stdout_happy_returncode_fails.
  • N3 — TASK_CLASSES models for agy were all the same high-tier string: changed all 5 entries to "tui-controlled" placeholder + docstring comment explaining the agy-specific exception (no model flag → per-class differentiation is meaningless).
  • N4 — registry ordering: accepted as cosmetic.

Operational note — `--mode danger requires --worktree` regression

My always-on-danger fixup tripped a small ergonomic regression: dispatch_smart.py at line 327 requires --worktree for any danger-mode dispatch (originally there to keep codex writes isolated). Forcing mode=danger for agy review-class dispatches means they now also require a worktree, even though agy in review mode doesn't write to disk. The 6-lane calibration tripped over this on cells 1 + 6 (read-only review dispatches) — re-fired with worktrees, worked fine. Follow-up: loosen the worktree-required check for agents that don't actually write under danger mode (codex stays as-is; agy gets a "no-write danger" carve-out).
