  • ds-flash (draft): produced YAML snippet with cooldown-policy: period: "7d". Fabricated schema — that's not in Dependabot's v2 reference.

Pattern: DeepSeek is unreliable on factual claims about GitHub-specific schemas/behavior. It synthesizes plausible-sounding YAML and security claims without verifying. Don't trust DS for review-class work on GH Actions / Dependabot / similar tooling.

3. DeepSeek shines on architecture (when fact-checking isn't required)

These were genuine architectural insights nobody else caught. DS is fine for design reasoning; just don't trust its factual claims about external systems.

4. dispatch_smart parser bug — false-positive rate-limit detection

4 of 5 architect dispatches were marked OK=False, resp_chars=0, RateLimitedError. Actually the models succeeded — dispatch_smart's _RATE_LIMIT_RE regex matched the phrase "rate limit" inside the model's actual response (because the architect prompt asks to discuss rate-limit failure modes). Bug applies to claude, deepseek (and possibly others). Only codex didn't trip it (different stderr handling). Real responses recovered via raw hermes calls bypassing the parser. Real bug — fix in scripts/agent_runtime/adapters/{deepseek,claude,grok,qwen}.py to require the "rate limit" pattern to appear in stderr/error contexts, not just any combined stdout+stderr.

5. codex (gpt-5.5) is inconsistent

Codex's R1 ground-truth run on PR #1333 caught all 3 findings (F1/F2/F3). Codex's R6 re-run of the same prompt caught only pip-pin (1 of 4). Possible reasons: model output variance, retry-on-empty differences, prompt-cache state. Implication: single-shot codex verdicts shouldn't be treated as definitive ground truth. Multi-agent consensus is more reliable.
