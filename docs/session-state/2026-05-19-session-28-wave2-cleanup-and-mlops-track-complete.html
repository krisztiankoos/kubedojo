#1332 — Incident dedup gate false positive

The gate uses regex catalogs in scripts/audit_incident_reuse.py. Two patterns relevant here:

r"AWS.{0,80}S3.{0,80}us-east-1"
r"S3.{0,80}us-east-1.{0,80}(2017|outage)"

Run with re.IGNORECASE | re.DOTALL, so . matches newlines. The variable definition

variable "aws_region" {
  type        = string
  description = "AWS region for the optional S3 exercise."
  default     = "us-east-1"
}

has "AWS" → "S3" → "us-east-1" all within 80 chars of each other. The teaching content has nothing to do with the 2017 us-east-1 S3 outage (canonical in platform/foundations/reliability-engineering/module-2.2-failure-modes-and-effects.md), it just happens to mention all three tokens in close proximity in a Terraform variable. Fix: change the default region to us-west-2 AND repaint "S3" to "bucket" in the description. Both changes are behavioral no-ops for the optional exercise — the IAM policy is region-agnostic, and the variable name still drives the AWS provider. Gate re-ran green.

This is a recurring failure mode worth a memory note for future Wave-N writers: variable descriptions in HCL that combine cloud-vendor name + service + region can trip incident-dedup regexes. Either (a) the writer brief should warn codex about this, or (b) the gate's region pattern should require a 2017-or-outage hint. Filed below as a follow-up.
