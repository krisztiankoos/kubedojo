Key decisions / contract changes

D1 — citation_backfill.py is now append-only

Two root causes diagnosed during the audit (live in scripts/citation_backfill.py at the time of the incident):

  1. REWRITE_DISPOSITIONS (soften_to_illustration, cannot_be_salvaged) caused substring-swap of body prose (line 1555 of pre-fix version). The seed JSON authorized the rewrites; the diff linter _verify_diff_is_additive trusted the seed. Codex's suggested_rewrite text routinely stripped concrete claims (numbers, names, hard requirements) to make them un-citable instead of finding a citation. Mechanism: module-1.2 had its "Marcus" narrative opening stripped but kept "He was trying..." as an orphan pronoun; concrete RAM/model-size table replaced with vague labels; duplicate Ollama-install sentence introduced.
  2. Sources rebuilt wholesale from seed, not merged. _strip_sources_section(new_body) removed the existing ## Sources heading AND everything after it (which is how module-1.9's ## Next Module heading vanished). Then _build_sources_section_from_seed appended only seed-listed URLs. Result: module-1.8 Sources 10 → 3; module-1.9 12 → 4 + broken nav chain.

Fix shipped in PR #1172: rewrite-disposition claims are now skipped (recorded as {status: "skipped", reason: "rewrites_disabled_pending_redesign"}); Sources are merged additively via new _merge_sources_section that parses existing URLs and only appends non-duplicates. Three regression tests plus a fail-closed-on-missing-jq test. Existing 6 tests + 4 new tests pass. Claude opus review: APPROVE_WITH_NITS (4 non-blocking nits).

D2 — First pre-tool-use hook is live (epic #1175)

PR #1177 shipped the block-branch-create hook. Combined with the _lib.sh extraction in PR #1179, the harness now has a working pattern for hooks 2-5. Lib functions: detect_primary_dir / normalize_path / is_inside_primary / read_tool_payload / deny. The denial template is 3-line: rule name + allowlist hint + memory-file reference.

Lessons learned (saved as memory)

L1 — dispatch_smart.py --worktree resolves relative to Bash cwd

Bash cwd leaks across tool calls (only shell state resets between calls). If an earlier cd .worktrees/X for a pytest run leaves the cwd inside a worktree, the next dispatch_smart.py --worktree .worktrees/X ... resolves to .worktrees/X/.worktrees/X and fails with FileNotFoundError. Always cd back to primary repo root before dispatching (or pass an absolute path). Verify with --dry-run first — the [dry-run] cwd=... and [dry-run] worktree=... lines reveal the doubled-path bug instantly. Hit on the PR #1172 review dispatch. Memory: feedback_dispatch_smart_cwd_trap.md.

Secondary note: dispatch_smart.py review --agent claude defaults to claude-sonnet-4-6, not opus. Pass --model claude-opus-4-7 explicitly when opus review is required. Captured in the same memory.

L2 — Untracked artifact + concurrent codex dispatch → divergence requires manual reconcile

The KodeKloud gap analysis HTML was created untracked on local main BEFORE codex was dispatched to augment it. Codex's worktree (based off a prior tracked commit) created its own version of the artifact from scratch (it had no tracked file to start from). When mid-session direction arrived to flip a section (Jenkins removal), the local untracked version diverged from codex's in-flight version. Recovery: apply the fixup edits in codex's worktree (after codex pushed), commit + push as a second commit on the same branch, delete the local untracked version. Pattern: for research artifacts that codex will modify, commit the initial version to the codex branch first, or accept the two-commit recovery shape.
