Decisions / contracts (durable)

Pattern B — overlap detection in apply_inject_plan

Contract: any inline_insertion whose anchor's byte offset in new_body falls inside an existing [label](url) region is treated as already addressed. The claim is recorded with status: "applied", note: "already_cited_in_body"; body unchanged. The coverage gate's applied_inline_ids filter (status == "applied") admits these as satisfying the claim. Idempotent across re-runs.

The detection helper _is_inside_existing_markdown_link(body, idx) mirrors the depth-aware paren parser style already used by _sanitize_sources_section_urls. Note: only detects inline ]( link syntax; does NOT detect reference-style links ([label][ref]). Not currently a problem since KubeDojo modules use inline links exclusively, but documented limitation.

Pattern C — codex-completeness soft-skip

Contract: when codex's inject plan omits a cited-disposition claim from BOTH inline_insertions AND skipped_claims, the orchestrator (not codex) marks the claim soft-skipped with reason: not_addressed_by_agent. Sidecar at .pipeline/citation-revisions/{slug}.json gains a codex_dropped array preserving claim_id, claim_text, span_hint, proposed_url, disposition for later human review. The inject result's new top-level codex_dropped_count field exposes the count for operators / briefing API.

The hard-fail rewrite_dispositions_not_addressed diff_issue is UNCHANGED — only the cited-coverage gate was relaxed. Rewrites still hard-fail.

Note: this masks a codex correctness problem (incomplete plans). The acceptable substitute behavior is "drop one claim, save it for human review, keep 15+ good wraps." Without this, one missing claim would have discarded an entire inject's work. If the codex_dropped_count trend across modules climbs, that signals a writer-prompt regression.
