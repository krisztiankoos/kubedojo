Read after user correction "did you research hermes docs and guides and tutorials?" — answer was no, gap closed.

  • 30+ inference providers wired in plugins/model-providers/ including gemini + google-gemini-cli (OAuth, free-tier supported) — contradicts the earlier "Gemini NOT via hermes" memory.
  • codex_responses_adapter.py:752-754: only passes max_output_tokens if caller provides it. No hermes-imposed default cap on grok at the adapter level.
  • agent/models_dev.py:407,495: defaults max_output_tokens to 8192 for unknown models. The models_dev_cache.json lists grok-4.3 with limit: {output: 30000} — 22k words. Hermes should send 30000 not 8192 for grok-4.3.

  • hermes -z stdin handling is finicky: cat prompt | hermes -z "$(cat -)" falls through to TUI mode (banner + tool list output, no model response). Correct invocation is hermes -z "$PROMPT" with prompt as direct arg. Confirmed empirically this session.
