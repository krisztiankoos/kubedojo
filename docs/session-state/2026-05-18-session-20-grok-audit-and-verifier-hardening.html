Why this session matters — the deterministic-gate validation

Session 19 produced a quality audit of grok-as-primary reviewer over 27 production PRs. The audit found a single false-negative (PR #1257 — grok hallucinated "nginx ships wget" as the APPROVE rationale on a lab that had kubectl exec ... -- wget against --image=nginx pods). The audit's central claim: this failure mode is harder to catch than a silent miss, because the reviewer engages with the dimension and confidently produces the wrong supporting fact. Recommended fix: a deterministic image-binary gate independent of any LLM reviewer.

This session built that gate (PR #1277) and validated it empirically. After landing:

  • Full-corpus scan against 261 k8s modules surfaced 5 modules with violations.
  • Of the 5, 3 are real bugs (CKA 1.2 / CKA 3.6 / CKS 0.3) — same #1257 bug class, never caught by the LLM reviewer cascade. Fixed in PR #1279.
  • Of the 5, 2 are acceptable pedagogy (module-2.1-pods has inline "# If bash not available" + fallback; module-3.2-seccomp has an explicit fallback paragraph). Gate fires correctly on the literal exec; modules handle the limitation editorially. Triaged as not-a-bug.

So the new gate independently surfaced and enabled the fix of 3 additional production lab bugs the LLM reviewer cascade had not caught before merging. 1 known bug + 3 latent bugs = 4× ROI on the audit work in a single autonomous run.
