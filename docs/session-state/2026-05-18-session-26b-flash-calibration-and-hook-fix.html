#1321 — hook fix detail

Bug: .claude/hooks/block-direct-commit-on-main.sh:27 used git -C "$CWD" rev-parse --abbrev-ref HEAD where $CWD = harness-reported cwd (workspace project_dir). When the actual git commit target was a worktree (via cd .worktrees/X && prefix or git -C /worktree flag), the hook checked the wrong branch. False-positive denied legitimate worktree-branch commits.

Hit twice in session 26:

  • Session-26 handoff commit attempt (handoff: session 26 ...) — passed only because the handoff: prefix is in the allowlist.
  • PR #1318 fix-pass commit (fix(content): remove invalid ce-datacontenttype...) — passed only because the subject happened to contain (#1309).

Fix: parser now walks tokens left-to-right, tracking cd <path> segments and the -C <path> flag on the git invocation. Branch check runs against the resolved effective tree, not the harness cwd. Pre-filter regex loosened to match git -C path commit form (was only matching git[space]commit directly).

21/21 tests pass: 14 existing + 7 new regression covering both directions (worktree commits pass; main commits still denied even when git -C /primary is used from elsewhere).
