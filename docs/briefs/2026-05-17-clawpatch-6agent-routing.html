4. Provider rotation policy (cross-family rule)

kubedojo's docs/review-protocol.md requires the reviewer to be a different model family from the author. clawpatch makes this trivially enforceable at the feature-slice level (finer than today's per-PR rule):

  • If a PR was written by codex → review the changed feature slice with acpx claude:claude-opus-4-7 or acpx gemini:gemini-3.1-pro-preview.
  • If a PR was written by claude (inline or headless) → review with codex or acpx gemini:*.
  • If a PR was written by gemini → review with codex or acpx claude:*.
  • For 3-way coverage on high-leverage merges (volume rewrites, audit-fix landings): run two clawpatch reviews on different slices with different providers. The persistent findings DB makes the second-pass cost ~30s + N codex-or-claude calls.
