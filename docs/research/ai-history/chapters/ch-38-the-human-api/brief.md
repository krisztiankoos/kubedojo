# Brief: Chapter 38 - The Human API

## Thesis
Machine learning requires massive, annotated datasets. Amazon created Mechanical Turk to solve its own internal catalog issues, transforming human cognitive labor into an API endpoint. This infrastructure allowed researchers to cheaply crowdsource the labels that would eventually train AI to replace that very labor.

## Scope
- IN SCOPE: Amazon Mechanical Turk (MTurk) launch, "Artificial Artificial Intelligence", the economics of micro-labor, early academic adoption (Snow et al. 2008).
- OUT OF SCOPE: Modern RLHF click-farms (belongs to Part 8/9).

## Scenes Outline
1. **The Catalog Problem:** Amazon struggles to find duplicate product listings using algorithms and builds an internal tool routing tasks to humans.
2. **Artificial Artificial Intelligence:** Jeff Bezos and team launch MTurk publicly as a web service API.
3. **The Annotation Engine:** AI researchers realize MTurk is the missing infrastructure for building massive datasets, shifting academic data labeling.
