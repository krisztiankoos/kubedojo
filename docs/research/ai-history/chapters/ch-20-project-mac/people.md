# People: Chapter 20 - Project MAC

- Robert M. Fano: founding Project MAC director; central voice for the computer
  utility vision and the "project, not laboratory" institutional style.
- Fernando J. Corbato: CTSS and Multics leader; anchors the systems side of
  Project MAC and the practical move from batch computing to time-sharing.
- J. C. R. Licklider: ARPA/IPTO visionary and later Project MAC director; use
  as funding/vision context, but do not make him the only protagonist.
- Edward Fredkin: Project MAC director after Licklider; Progress Reports VIII/X
  frame the early-1970s reorganization and automatic-programming direction.
- Marvin Minsky: AI group leader; connects Project MAC infrastructure to vision,
  robotics, symbolic AI, and the later MIT AI Lab.
- Seymour Papert: AI group figure; use for Project MAC AI community and later
  perceptron/Logo context only where anchored.
- Daniel G. Bobrow: LISP 2 / AI programming-language work; useful for language
  infrastructure, not a main character.
- Carl Hewitt: PLANNER; important for automatic programming and knowledge-based
  programming as handoff to expert-system-era symbolic software.
- William A. Martin and Joel Moses: MATHLAB/MACSYMA lineage; use to show
  symbolic manipulation as a practical AI-adjacent system.
- Jon L. White, Guy Steele, Stavros Macrakis, David Moon: MACLISP/MACSYMA
  infrastructure figures; mention only if prose needs concrete system builders.
- Donald Eastlake: ITS 1.5 reference manual; useful as the named systems anchor
  for the AI time-sharing environment.
- Tom Van Vleck: participant historian; use his Multicians page as a guide and
  secondary check, not as the sole anchor for contested claims.
