## Навіщо це існує

Безкоштовний текстовий навчальний план для вивчення Kubernetes та платформної інженерії.

- **Безкоштовно** — Без платних стін, відкритий код
- **Теорія на першому місці** — Зрозумійте принципи до інструментів
- **Текстовий формат** — Зручний пошук, контроль версій, без відео

**Чим ми НЕ є:** Заміною платних курсів, як-от KodeKloud чи Udemy. Ми не пропонуємо пробних іспитів, відеоуроків чи <!-- VERIFY: лабораторних робіт -->практичних завдань для кожного модуля. Для симуляції іспиту використовуйте [killer.sh](https://killer.sh). Для інтерактивних лабораторних — [killercoda.com](https://killercoda.com).

## Філософія

**Теорія перед практикою.** Ви не зможете усунути несправність того, чого не розумієте.

**Без зубріння.** Документація K8s доступна під час іспитів. Ми навчаємо навігації, а не запам'ятовування YAML.

**Принципи понад інструменти.** Інструменти змінюються. Основи — ні. Вивчайте і те, і те, саме в такому порядку.

## Як долучитися

**Що нам потрібно:**
- **Практичні вправи** — Реальні сценарії, а не іграшкові приклади
- **Історії з продакшну** — Інциденти, що навчають
- **Експертиза з інструментів** — Глибокі огляди ArgoCD, Prometheus, Vault тощо
- **Виправлення помилок** — Друкарські помилки, застарілі команди, некоректний YAML

**Що ми НЕ створюємо:**
- Симулятори іспитів — Використовуйте [killer.sh](https://killer.sh) (входить до покупки іспиту)
- Лабораторні середовища — Використовуйте [killercoda.com](https://killercoda.com) або локальний kind/minikube
- Відеоконтент — Текстовий формат, зручний пошук, контроль версій

**Як долучитися:**
- Відкрийте Issue для обговорення перед великими PR
- Дотримуйтесь наявної структури модулів
- Перевіряйте всі команди та YAML перед надсиланням
