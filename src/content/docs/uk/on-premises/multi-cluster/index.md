Організації, що використовують on-premises рішення, рідко обмежуються одним кластером Kubernetes. У міру зростання команд та диверсифікації навантажень виникає потреба у кількох кластерах — для розділення dev/staging/prod, регіональних розгортань, ізоляції орендарів (tenants) або просто для зменшення радіусу ураження при збоях. Але на відміну від хмари, де створення нового кластера займає хвилини і коштує лише викликів API, on-premises мультикластерність означає управління фізичними серверами, розміщенням площини управління та автоматизацією життєвого циклу з обмеженим обладнанням.

Цей розділ охоплює інфраструктурні платформи, на яких базується Kubernetes (vSphere, OpenStack, Harvester), стратегії площини управління, які дозволяють запускати багато кластерів на кількох серверах (vCluster, Kamaji), та декларативні інструменти життєвого циклу, що ставляться до кластерів як до «худоби» (Cluster API на bare metal).
